# 🌐 웹 사용법

평소 매물 / 만남 / 매칭 관리는 모두 여기에서. 모바일 폰에서도 잘 동작합니다 (하단 탭바).

## 💡 꿀팁 필독: 홈 화면에 추가하기 (앱처럼 쓰기)

폰에서 매번 URL 칠 필요 없이 **홈 화면에 핑크 하트 아이콘 💗 추가**해두면 일반 앱처럼 켜기/끄기 가능.

![홈 화면에 추가 단계별](/static/docs/ios_addtohome.svg)

### iOS (사파리)
1. 사파리에서 meetcute URL 접속 (예: `https://abc.trycloudflare.com`)
2. 하단 가운데 **공유 버튼 ⬆️**
3. 스크롤 → **"➕ 홈 화면에 추가"**
4. 이름 확인 → **추가**

### Android (크롬)
1. 크롬에서 URL 접속
2. 우상단 ⋮ → **"홈 화면에 추가"**

### 일반 책갈피랑 뭐가 다른가요?
- URL 바가 안 보임 (풀스크린)
- 핑크 하트 💗 아이콘
- 앱 스위처에 따로 카드로 떠서 다른 앱 전환 빠름
- 모바일 다크모드 / 세이프에어리어 자동 인식

### ⚠️ 알아둘 점
- URL 이 바뀌면 (cloudflared 재시작 시 quick mode 등) 아이콘은 옛 URL 로 가서 "사이트 못 찾음" 뜸
- **URL 바뀌면 텔레그램으로 자동 알림 옴** — 그때 새 URL 로 다시 "홈 화면에 추가" 하면 됨
- 영구 URL 쓰려면 named tunnel 셋업 (운영 가이드 → 외부 노출 참조)

## 2. 매물(=소개해줄 사람) 등록

### 등록 흐름
1. 매물 목록 → **+ 새 매물**
2. **성별** (등록 후 변경 불가 — `public_id` 발급 기준)
3. 필수 5칸: 이름·사는곳·직장·나이·키
4. 사진 1~5장 (jpg/png/webp/heic — EXIF 자동 회전)
5. 이상형 / 메모는 "추가 정보 (선택)" 펼쳐서 입력 또는 등록 후 수정에서

### ID 규칙
- **M-001 / F-001 / X-001** 형식으로 자동 발급
- 같은 성별 내에서 max+1, 삭제해도 재사용 안 됨

### 이름(메모용) — 다른 마담뚜한테 안 보여요
- "이름" 필드는 본인이 부르기 편한 메모용
- **owner 본인과 책임자만 표시** (다른 마담뚜 화면엔 public_id 만)
- 실명 그대로 박아도 무방 (다른 사람한텐 안 보임)

## 3. 매물 상태 (자동 계산)

|뱃지|판단 기준|
|---|---|
|🟢 소개 가능|만남 기록 없거나 모두 종료|
|🟡 진행 중|PENDING/CONTINUING 만남 1개 이상|
|💗 매칭됨|MATCHED 만남 1개 이상|

상태는 **저장 X · 항상 만남 기록으로부터 자동 계산**. 매물 수정으로 직접 못 바꿈.

## 4. 만남 기록 (Encounter)

### 새 만남
- `/encounters/new` 또는 매물 상세의 **+ 새 만남**
- 두 매물 선택 + 만난 날짜 + 결과(PENDING 기본) + 메모

### 결과 종류
| 값 | 의미 | 상태 영향 |
|---|---|---|
| PENDING | 만남 잡힘 / 결과 미정 | → 진행 중 |
| CONTINUING | 계속 만나는 중 | → 진행 중 |
| MATCHED | 매칭 성공 (사귀기로) | → 매칭됨 |
| ENDED_A | A가 거절 | (이력만) |
| ENDED_B | B가 거절 | (이력만) |
| MUTUAL_END | 양쪽 다 노 | (이력만) |

### 결과 변경 이력 (📜 자동)
outcome 이 바뀔 때마다 자동 이벤트 1건씩 기록 — 매물 상세 / 만남 편집 페이지에 시간순 타임라인.
- 변경 사유 메모를 입력란에 짧게 적으면 같이 박힘
- notes-only 수정은 이벤트 안 만듦 (소음 방지)
- 누가 바꿨는지 닉네임으로 기록

> 예: "PENDING (최초 등록) → CONTINUING (1차 잘됨) → MATCHED (사귀기로 함)" 같은 스토리가 시간순으로 쌓임.

## 5. 매물 활동 추적

매물 상세에 자동 표시:
- **총 만남 / 매칭 / 진행 중 / 마지막 활동** 카드
- **만남 이력 타임라인**
- **📝 정보 변경 이력** — age/사는곳/직장/키/이상형/메모/이름 수정 시 자동 스냅샷, before→after 색깔 표시, 편집자 닉네임

### 매물 목록 필터
상단 `/persons` 필터:
- **활동**: 진행 중 / 30일+ 미활동 / 한 번도 만남 X
- **정렬**: 등록순 / 최근 활동순 / 오래 잠자는순
- **owner**: 내 매물 / 미지정 / 특정 마담뚜

## 5.5 매물 공개 범위 🔒

매물 등록한 사람(owner)이 매물 수정 화면에서 공개 범위를 정할 수 있어요:

- **모두 공개 (기본)** — 모든 마담뚜가 볼 수 있음
- **특정 마담뚜만** — 선택한 마담뚜 + 나 + 책임자만 접근 가능

비공개로 설정하면:
- 매물 목록에서 제외되어 보임 (허락 안 된 admin)
- 매물 상세 페이지 403
- 매물 상세 헤더에 🔒 비공개 뱃지 (owner / 책임자에게만 표시)

> 공개 범위 변경은 **owner 본인 또는 책임자만 가능**. 다른 마담뚜는 설정 UI 자체 안 보임.

## 6. 다른 마담뚜의 매물에 소개 요청 보내기 📨

같이 쓰는 마담뚜가 2명+ 일 때:
1. 다른 마담뚜 소유 매물 상세 → **📨 소개 요청** 버튼
2. 내 매물 중 어느 것 소개? + 메모 → 보내기
3. 상대 마담뚜가 `/requests` 에서 확인 → 본인이 IRL로 매물에 물어봄
4. **수락** → 만남 기록(PENDING) 자동 생성, 양쪽 매물 타임라인에 즉시 노출
5. **거절** → 사유 메모와 함께 보낸이에게 알림

요청 상태: PENDING / ACCEPTED / DECLINED / WITHDRAWN

> 자기 매물끼리 매칭은 요청 X — 그냥 `/encounters/new` 에서 두 매물 골라 바로 기록.

## 7. 호환성 체크

`/compatibility` 또는 매물 상세의 **호환성 체크**:
- 두 매물 선택
- 이전 만남 기록 / 성별 / 나이 차이 / 키 차이 분석
- "이 둘로 만남 기록 만들기" 버튼으로 바로 등록

## 8. 검색

매물 목록 검색창: **ID / 이름 / 지역 / 직장** 부분 일치.

## 9. 매물 수정 권한

- **등록한 admin (owner) 만** 수정/삭제
- 책임자(👑)는 모든 매물 수정/삭제 (override)
- 미지정 매물은 누구나 (담아가게)
- 권한 없는 사람에겐 🔒 표시

## 10. 삭제 정책

### 매물 삭제 → 하드 삭제
- DB row + 사진 파일 + **정보 변경 이력** 영구 제거
- **만남 기록은 보존** — FK 끊고 `public_id (deleted)` 스냅샷으로 취소선 표시
- 한 번 삭제하면 복구 X. 보관할 거면 그냥 두세요.

### 만남 기록 삭제
- 잘못 입력한 경우만
- 결과 변경 이력(EncounterEvent)도 함께 삭제
